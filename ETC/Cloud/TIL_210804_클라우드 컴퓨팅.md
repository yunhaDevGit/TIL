# 클라우드 컴퓨팅 아키텍처 분석

### 클라우드 컴퓨팅

- 인터넷 기반 컴퓨팅의 일종으로 정보를 자신의 컴퓨터가 아닌 인터넷에 연결된 다른 컴퓨터로 처리하는 기술
- 클라우드 = 인터넷 , 컴퓨팅 = 컴퓨터를 가지고 하는 일(개발, 게임...) 
  - 인터넷을 기반으로 하는 컴퓨터를 가지고 하는 일을 하는 것



#### 클라우드 컴퓨팅 특징

- On-demand self-service (self-provisioning)
  - 요청이 있을 때마다 스스로 서비스를 제공하는 것
  - 사업자와 직접 상호 작용하지 않고, 사용자의 개별 관리 화면을 통해 서비스를 이용할 수 있다
- Broad network access 
  - 광범위한 네트워크 접근성 
  - 모바일 기기 등 다양한 디바이스를 통해 서비스에 접속할 수 있다
    - 지역 뿐만 아니라 다양한 디바이스도 제공
- Resource pooling
  - 자원 풀링 : multi-tenant model
- Rapid elasticity
  - 신속한 확장성, 자동화
  - 필요에 따라 필요한 만큼 Scale Up & Down이 가능하다
- Measured service(metering)
  - 이용한 만큼 요금이 부가되는 종량제 (pay as you go) 



### 클라우드 컴퓨팅 모델

- Service Models
  - Software as a Service(SaaS)
  - Platform as a Service(PaaS)
  - Infrastructure as a Service(IaaS)
- Deployment Models
  - Private Cloud
  - Community Cloud
  - Public Cloud
  - Hybrid Cloud





**Q. 클라우드 컴퓨팅과 호스팅 서비스와의 가장 큰 차이점은?**

- 호스팅 서비스는 호스팅 업체의 서버 중 일부를 임대해서 사용하는 반면 클라우드 컴퓨팅은 호스팅 업체의 가상 서버를 단독으로 생성하여 사용할 수 있다.
- 호스팅 서비스는 자원 사용량이 제한되어 있고, 서버 관리 권한이 없는 반면 클라우드 서비스는 사용자가 원하는 양의 자원을 사용할 수 있다.
